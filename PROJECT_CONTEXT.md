# IceZhou Portfolio Project Context

这个文件是后续任务的优先上下文入口。每次继续修改前，先读这里，再按需查看 `src/main.jsx` 和 `src/styles.css`，可以减少依赖长聊天历史。

## 项目概况

- 项目类型：设计师作品集网站，React + Vite。
- 本地预览：`http://127.0.0.1:5174/`。
- 主要代码：`src/main.jsx`。
- 主要样式：`src/styles.css`。
- 静态资源：`public/assets/`。
- 字体资源：`public/fonts/`，已倾向本地托管字体，避免外部字体网络加载失败。
- 常用命令：`npm run dev -- --host 127.0.0.1 --port 5174`，`npm run build`。

## 工作原则

- 不要还原或覆盖用户已有改动，当前工作区经常是 dirty 状态。
- 视觉调整优先小步修改，改完要用本地预览核对。
- 图片和视频素材不要直接塞进 JS bundle，放在 `public/assets/` 下按目录组织。
- 大量图片页面需要考虑加载速度：展示图用压缩版本，预览模式用高清版本。
- 后续部署面向中国大陆稳定访问，推荐阿里云 OSS + CDN 托管静态资源，网站本体走静态构建部署。

## 部署建议

最佳方向：

1. React/Vite 继续作为静态前端。
2. 项目数据先抽成结构化 JSON/JS 数据文件，降低 `src/main.jsx` 复杂度。
3. 图片和视频上传到阿里云 OSS，开启 CDN。
4. 页面中只引用 CDN URL，图片启用懒加载。
5. 大图生成展示版与预览版两套尺寸。
6. 如果更新频率较低，暂时不需要完整 CMS；可以先用本地数据文件 + OSS 管理素材。
7. 后续若频繁新增项目，再考虑轻量 CMS 或自建后台。

## 后续优化优先级

1. 抽离作品数据和图片清单，减少 `src/main.jsx` 体积。
2. 建立资产命名规范：`project-slug/display/01.webp`、`project-slug/preview/01.webp`。
3. 增加构建前资源检查脚本，避免图片路径错误。
4. 为当前稳定版本建立 Git 分支和里程碑提交。
5. 部署前统一压缩图片、转 WebP/AVIF，并接入 OSS + CDN。
