## 布局快照

该模块位于【管理 - 用户管理】页面中，基于双栏布局（布局 1）。功能入口为用户管理页面的【自动化配置】和【自动化日志】按钮。核心配置通过 Modal Dialog 承载，日志通过独立页面承载。

**布局类型**：布局 1（双栏）— Navsidebar + Content
**页面基准**：1920×1080（设计稿主画板 1912×1035）

**主要区域划分**：

**特殊约定**：

- 配置侧边栏（204px）为左侧模块导航，包含多级 Tab 分组
- 自动化配置通过弹窗承载，非独立页面
- 自动化日志为独立子页面，通过按钮跳转
- Dialog body 高度随配置内容动态变化
- 规则配置区域最大高度 308px，超出后内部滚动
- v2.0 内包人员规则行为 3 字段（部门/项目 + 权限角色 + 小组），较 v1.0 入职规则多一个字段

## 核心交互链路

### 主链路 1：入职自动化配置（v1.0 已有）

```
1. 用户管理页面 → 点击「自动化配置」按钮 → 弹窗打开（Modal Dialog 720px）
2. 弹窗内「新入职人员」区域 → Switch 开关切换 → 启用"新入职人员自动加入米线"
3. 开关开启后 → 显示规则配置区域（按部门架构分配权限角色）
4. 点击「添加条件」 → 新增一行规则条目（部门架构 Select + 权限角色 Select）
5. 选择部门架构 → 树形下拉菜单（支持多选，搜索时子级命中展示父级）
6. 选择权限角色 → 下拉菜单（支持多选，数据来源为当前项目已配置的权限角色）
7. 底部固定规则"未在设定范围内的其他所有用户分配为" → 选择权限角色（含「不分配权限角色」）
8. 点击「保存」 → 校验所有条目完整性 → 保存成功关闭弹窗 + Toast 反馈
```

### 主链路 2：离职/转岗自动化配置（v1.0 已有）

```
1. 弹窗内「离职/转岗人员」区域 → Switch 开关切换 → 启用"离职/转岗员工自动处理"
2. 开启后说明文字：离职/流转人员的状态将修改为"已离职"/"已转岗"
3. 点击「保存」 → 配置保存 + Toast 反馈
```

### 主链路 3：内包人员自动加入配置 [v2.0 新增]

```
1. 弹窗内「内包人员」区域 → Switch 开关切换 → 启用"内包人员自动加入米线"
2. 开关开启后 → 显示规则配置区域（按部门/项目分配权限角色和小组）
3. 点击「添加条件」 → 新增一行规则条目：
   - 原属部门/项目 Select（多选）
   - 权限角色 Select（多选）
   - 小组 Select（多选）
4. 底部固定保底规则"未在设定范围内的其他所有内包人员分配为"：
   - 选择权限角色（含「不分配权限角色」选项）
   - 选择小组（当权限角色选择「不分配权限角色」时，小组 Select 置灰禁用）
5. 点击「保存」 → 校验 → 保存成功关闭弹窗 + Toast 反馈
```

### 主链路 4：内包人员退出自动处理 [v2.0 新增]

```
1. 弹窗内「内包人员」区域 → 第二个 Switch → 启用"内包人员退出自动处理"
2. 开启后说明文字：内包结束的人员状态将修改为"内包结束"
3. 点击「保存」 → 配置保存 + Toast 反馈
```

### 主链路 5：自动化日志查看 [v2.0 新增]

```
1. 用户管理页面 → 点击「自动化日志」按钮 → 跳转日志页面
2. 筛选栏：时间范围（DateRangePicker）+ 操作类型（全部/自动加入/自动退出）+ 执行结果（全部/成功/失败）+ 人员姓名搜索
3. 表格展示日志列表：操作时间、人员姓名、操作类型、执行详情、执行结果、失败原因、操作
4. 失败记录 → 操作列显示「重试」按钮 → 点击重试 → loading + 结果更新 + Toast
5. 分页：默认 20 条/页
```

### 辅助链路

- 规则条目删除：点击条目右侧删除图标移除该条规则（保底规则不可删除）
- 条目校验失败：保存时必填字段缺失的条目高亮提示，阻止保存
- 多条规则叠加：员工部门匹配多条规则时，所有命中规则的权限角色叠加分配
- 内包保底规则联动：选择「不分配权限角色」时，小组 Select 自动置灰
- 日志空态：无记录时显示"暂无自动化操作记录"

## 业务实体

### 核心实体：自动化配置

```yaml
entity: "AutoTransferConfig"
description: "项目级人员自动流转的规则配置，包含入职、离职/转岗、内包三个场景的自动化开关和规则"
```

### 字段清单

| 字段 | 类型 | 必填 | 说明 | 对应组件 |
|------|------|------|------|---------|
| onboard_enabled | boolean | 是 | 新入职人员自动加入米线开关 | Switch |
| onboard_rules | array | 否 | 入职自动分配规则列表（开关开启后显示） | 动态表单行 |
| onboard_rules[].departments | tree-select | 是 | 部门架构节点（多选，树形） | TreeSelect |
| onboard_rules[].roles | multi-select | 是 | 权限角色（多选） | Select |
| default_role | multi-select | 是 | 未匹配规则的默认角色分配（含「不分配权限角色」选项） | Select |
| offboard_enabled | boolean | 是 | 离职/转岗员工自动处理开关 | Switch |
| outsource_onboard_enabled | boolean | 是 | 内包人员自动加入米线开关 [v2.0] | Switch |
| outsource_rules | array | 否 | 内包人员自动分配规则列表 [v2.0] | 动态表单行 |
| outsource_rules[].departments | multi-select | 是 | 原属部门/项目（多选） [v2.0] | Select |
| outsource_rules[].roles | multi-select | 是 | 权限角色（多选） [v2.0] | Select |
| outsource_rules[].groups | multi-select | 是 | 小组（多选） [v2.0] | Select |
| outsource_default_role | multi-select | 是 | 内包未匹配规则的默认角色（含「不分配权限角色」） [v2.0] | Select |
| outsource_default_group | multi-select | 否 | 内包未匹配规则的默认小组（角色为「不分配」时禁用） [v2.0] | Select |
| outsource_offboard_enabled | boolean | 是 | 内包人员退出自动处理开关 [v2.0] | Switch |

### 关联实体

| 字段 | 类型 | 必填 | 说明 | 对应组件 |
|------|------|------|------|---------|
| requirement_type | enum | 是 | 需求类型 | Select |
| parent_requirement | reference | 否 | 父需求 | Select |
| assignee | user | 否 | 负责人 | Select |
| qa | user | 是 | QA | Select |
| dev_period | enum | 否 | 开发期 | Select |
| estimated_effort | number | 否 | 预估工时（人天） | Select |


| 字段 | 类型 | 必填 | 说明 | 对应组件 |

## 设计决策记录

| 版本 | 变更 | 原因 |
|------|------|------|
| v1.0 | 自动化配置通过 Dialog 承载而非独立页面 | 配置项相对简单，弹窗足以承载；减少页面跳转 |
| v1.0 | 入职规则采用「开关 + 动态规则列表」模式 | 不同项目自动化需求不同，需灵活配置；开关默认关闭降低误操作风险 |
| v1.0 | 离职/转岗开关默认开启 | 离职/转岗自动标记是普遍需求，默认开启减少配置成本 |
| v1.0 | 部门架构选择使用树形下拉而非平铺 | 部门层级较深，树形结构更易定位；支持搜索提升效率 |
| v1.0 | 固定保底规则"未在设定范围内的其他用户" | 确保所有新入职人员都有明确的权限分配策略，避免遗漏 |
| v1.0 | 离职/转岗人员头像标记（设计侧建议） | 前台列表中离职标签会影响页面观感，在头像上体现更轻量 |
| v2.0 | 新增「内包人员」配置区域 | 内包场景自动化需求增长，需独立配置入口 |
| v2.0 | 内包规则行增加「小组」字段（3字段） | 内包人员需同时分配权限角色和小组，较正式员工多一个维度 |
| v2.0 | 保底规则「不分配权限角色」联动禁用小组 | 无权限角色时分配小组无意义，避免用户误操作 |
| v2.0 | 保持 720px Dialog 宽度不变 | 3字段规则行经计算可在 680px 内容宽度中容纳（300+170+170+gap），无需扩宽 |
| v2.0 | 新增自动化日志独立页面 | 日志数据量大、需筛选和分页，弹窗无法承载，需独立页面 |
| v2.0 | 日志页复用双栏布局 | 与用户管理页面保持一致的导航结构和视觉层级 |

## 已知约束与 Gap

| 约束 | 说明 | 影响 |
|------|------|------|
| HRG 系统对接未定稿 | 入职/流转/离职/内包的触发时机和通知机制需与 HRG 团队确认 | 自动化处理的实际触发逻辑可能调整 |
| 环境-项目映射硬编码 | 每个生产环境对应固定的公司一级架构项目 | 新项目接入需更新映射关系 |
| 性能时效要求 | 入职/离职/转岗/内包处理需在当天 24:00 前完成 | 需要定时任务或事件驱动机制保证 |
| 规则区域高度限制 | 规则配置区域最大 308px | 大量规则时依赖内部滚动，可能影响操作体验 |
| 转岗回流无额外逻辑 | 离职后回流不做特殊处理 | 需要管理员手动恢复权限 |
| 日志重试机制待定 | 重试的具体执行策略（单条/批量、超时时间）待确认 | 可能影响重试按钮的交互细节 |
| 内包人员退出不设规则 | 内包退出仅标记状态，不支持按部门差异化处理 | 如有差异化需求，后续版本可能扩展 |

## 消费方式

### AI 生成新页面时

1. 读取「布局快照」确认双栏布局 + Dialog 弹窗模式 + 日志独立页面
2. 读取「核心交互链路」理解开关驱动的动态表单逻辑和日志筛选逻辑
3. 读取「业务实体」确定规则配置的数据结构（注意 v2.0 内包规则为 3 字段）
4. 读取「Figma 设计 Token」确保暗色主题下的精确色值和尺寸
5. 读取「弹窗尺寸规格」和「规则行布局」确保弹窗各状态下的精确尺寸
6. 读取「状态与空态」补全规则为空、校验失败、保底规则联动等非 happy-path 设计

### AI 扩展已有模块时

1. 先读取本文件，理解入职/离职/内包三个场景的配置结构
2. 若新增场景（如调岗独立配置），优先在现有 Dialog 的分组结构中寻找承载位置
3. 规则条目结构可作为新规则类型的模板复用（2字段版或3字段版）
4. 日志页面的筛选+表格模式可复用于其他操作日志场景
5. 变更后更新本文件的对应段落
