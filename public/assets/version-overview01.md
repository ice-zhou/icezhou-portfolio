# 版本总览（Version Overview Module）

> **Figma 源文件**：[米线标准稿 - Version Overview Module](https://www.figma.com/design/K3ow0taJSUaEo65k8HcOqB/%E7%B1%B3%E7%BA%BF%E6%A0%87%E5%87%86%E7%A8%BF?node-id=5059-31771)
> **模块名称**：主页面【版本总览】
> **模块目标**：随着版本+时间线逻辑的改动，增加了版本日历的概念，前台界面中需相应进行调整
> **当前参考基线**：全新功能
> **本次整理范围**：1.0 版本

## 1. 页面概述

版本总览是米线项目管理工具中的**核心模块**，用于展示和管理项目的版本日历、里程碑时间线和版本进度。用户可以通过该模块：

- 查看多个版本日历的时间线
- 跟踪里程碑节点（如白盒、开发截止、版本发布）
- 管理关注的版本日历
- 设定当前版本

**布局类型**：页面浮层（Overlay），版本总览位于整个页面导航栏内，hover 导航栏上的标签可唤起"版本总览浮层"

## 3. 功能清单

### 功能编号索引（对应 Figma 标注编号）

### 3.1 版本日历 Tab 栏（`*selectcard*` 组件）
- 使用米线 `*selectcard*` 组件展示用户关注的所有版本日历对象
- 每个日历项使用 `.selectcard-item` 子组件（含 mi-icon-circle-plus-outline 图标 + 日历名称）
- 点击 Tab 切换不同版本日历
- **Tab 过多处理**：当 Tab 数量超过浮层宽度时，出现左右翻页按钮组 (`arrow-left` / `arrow-right`，各 24×24px)
- 右侧 ⚙ 设置按钮 (28×28px，mi-icon-setting 图标)；hover 时出现 Tooltip "选取关注版本日历" (136×36px)
- 右上角 ✕ 关闭按钮 (main/close, 16×16px) 关闭浮层

### 3.2 配置关注版本日历（弹窗）
- **弹窗结构**：dialog 组件，整体 720×610px
  - `dialog-head` (720×54px)：标题"请选取关注的版本日历" + ✕ 关闭按钮
  - `dialog-body` (720×484px)：左右分栏
  - `dialog-foot` (720×72px)：底部操作区
- **左侧列表** (340×460px)：展示当前用户可见的所有版本日历（可勾选）
- **右侧列表** (340×460px)：展示已勾选（关注）的版本日历列表
- **默认关注**：用户默认设置为关注的日历，左侧默认勾选且保持禁用态；右侧固定显示这些日历
- **排序**：右侧界面中，用户可拖拽调整展示顺序
- **保存逻辑**：点击确认保存；点击取消放弃修改，需有二次确认提示
- **数据统计**：底部显示"表格数据量统计"

### 3.3 版本日历滚动轴（功能 ③）
- 进入日历界面后**默认定位显示当前版本**
- 版本列表纵向排列（tab 区域 118×437px），里程碑横向时间轴展示
- 每个版本行 102×36px，包含版本号文本 + mi-icon-view 可见性图标 (12×12px)
- 滚动吸顶处理（版本标题栏吸顶）
- Content 区域含滚动条 (Component 351, 8×101px)

### 3.4 设为当前版本（功能 ④）
- 每个版本标题栏 (478×45px) hover 后出现"设为当前版本"按钮 (112×24px，含 aim 图标 16×16)
- 点击后将该版本日历的"当前版本"设定为所选版本
- 当前版本标题栏中按钮变为"当前版本" (84×24px)，非 hover 态常驻显示
- 当前版本前端展示逻辑沿用旧版本，版本列表中标记为 `X.X(当前)`

### 3.5 里程碑卡片（list 组件）
- 每个里程碑卡片为 list 组件 (438×36px)，包含：
  - **名字区域** (218×36px)：如"4.16 LD白盒"、"4.16 开发截止时间"、"4.16 版本发布时间"
  - **标签区域** (100×36px)：精英 (40×24) / 特殊 (40×24) / Boss (44×24) 标签 + 距今天数指示器 (Frame 2036083691, 20×20)
  - **距今天数**：如 ">99天前"、"10天前"、"30天后"
  - **日期区域** (120×36px)：具体日期（如"2023/10/20"、"2024/03/10"、"2023/05/20"）+ 标签复显
- **底部装饰**：版本标记 (V1、V3 等)，用连线指向对应里程碑

### 3.6 收起/展开
- 版本总览浮层可收起为 Milestone Bar (289×27px)
- 收起时入口展示为"当前版本距离今天最近的时间线"
- Milestone Bar 结构：illustrations (27×27) + text 区域（name 233×22 + time 53×22）+ 操作按钮区 (92×32, 含删除/编辑/提醒图标)
- 展示示例："4.6版本建模完成最多字数最长情况... 30天后"

### 3.7 主页外显样式（功能 ①，老逻辑）
- Milestone 组件嵌入 Title Bar 导航栏中 (223×28px)
- 结构与收起态 Milestone Bar 一致：illustrations + name + time + 操作按钮
- 操作按钮区 (92×32)：删除 (mi-icon-delete) + 编辑 (edit-outline) + 提醒 (mi-icon-bell)

### 3.8 全局版本搜索（v1.1 新增，功能 ⑤）

#### 3.8.1 搜索入口
- **入口位置**：浮层吸顶区域 (640×54)，位于标题"版本总览"右侧、⚙ 设置按钮左侧
- **搜索组件**：使用米线 `search` 组件，与 Title Bar 中全局搜索风格一致
- **默认态**：search-box (200×28)，圆角 4px，背景 bg-level3，边框 border 色
  - 左侧 mi-icon-search (14×14) + placeholder 文案"搜索版本、里程碑..."
  - 右侧 clear-btn (14×14 圆形)，默认 opacity:0，有输入时显示
- **聚焦态**：search-box 宽度动画展开至 280×28 (200ms ease)
  - 边框变为 accent 色 (#409eff) + 2px 外发光 (rgba(64,158,255,0.1))
  - 背景变为 bg-level2（更透明，区分输入态）

#### 3.8.2 搜索范围
- 跨所有用户关注的版本日历
- 匹配字段：版本号、里程碑名称、标签（精英/特殊/Boss）
- 输入时 **300ms 防抖**后触发搜索

#### 3.8.3 搜索结果下拉面板（search-dropdown）
- **尺寸**：480×auto (max-height 420px)，从搜索框正下方弹出
- **圆角**：6px；**阴影**：0 6px 16px rgba(0,0,0,0.3)
- **结构**：
  - `search-dropdown-header` (480×32)：结果计数 "找到 N 个结果" + 快捷键提示 (↑↓ 选择 / Enter 跳转 / Esc 关闭)
  - `search-results-body`：可滚动结果列表区
  
#### 3.8.4 结果分组展示（result-group）
- 按**版本日历**分组，每组之间以 1px border 分隔
- **分组标题** `result-group-title` (—×24)：日历色点 (8×8 圆角方块) + 日历名称 + 结果计数（如"3 项"）
- **结果行** `result-item` (—×40)，每行包含：
  - 版本号 `ri-version` (36px 宽)：如"4.16"、"5.1"
  - 里程碑名称 `ri-name`：关键词命中部分用 accent 色 + 加粗高亮（如"4.16 LD**白盒**"）
  - 标签 `ri-tags`：复用里程碑标签组件（精英/特殊/Boss）
  - 日期+距今 `ri-meta` (72px 宽)：上行具体日期、下行距今天数
- **hover/selected 态**：背景变为 bg-hover (#333)
- **键盘导航**：↑↓ 键在结果行间移动选中态，Enter 确认跳转

#### 3.8.5 结果交互
- 点击某条结果 → 自动执行：
  1. 关闭搜索面板
  2. 切换到对应版本日历 Tab（`*selectcard*` 组件联动）
  3. 自动滚动定位到对应版本行
  4. 对应版本标题栏短暂闪烁高亮 (300ms)

#### 3.8.6 空结果态
- 使用 `search-empty` 组件 (480×170)
- 图标 (64×64 圆形背景 + 🔍 搜索图标) + "未找到相关版本内容" 文案
- 面板自适应收缩高度

#### 3.8.7 加载态
- 三点脉冲动画 (6×6 圆点 × 3，间距 6px，1.2s 循环) + "搜索中..." 文案
- 面板保持最小高度，避免布局跳动

#### 3.8.8 快捷键
| 快捷键 | 功能 |
|--------|------|
| `↑` / `↓` | 在结果行间移动选中态 |
| `Enter` | 确认跳转到选中的结果 |
| `Esc` | 关闭搜索面板，清空搜索框 |

## 4. 交互说明

### 4.1 浮层唤起/关闭流程
```
用户 hover 导航栏上的"版本总览"标签
  → 唤起版本总览浮层
  → 默认展示用户首个关注的版本日历
  → 默认定位到当前版本

用户点击浮层右上角 ✕ 关闭按钮 (main/close)
  → 关闭浮层
```

### 4.2 日历切换流程
```
用户点击 *selectcard* 组件中的日历 Tab 项
  → 切换到对应版本日历的时间线视图
  → 默认定位到当前版本

日历 Tab 过多时：
  → 出现 ◄ ► 翻页按钮 (arrow-left / arrow-right)
  → 用户点击翻页按钮查看更多日历
```

### 4.3 配置关注日历流程
```
用户 hover ⚙ 设置按钮
  → 出现 Tooltip "选取关注版本日历" (136×36)
用户点击 ⚙ 设置按钮
  → 弹出配置弹窗 (dialog 720×610)
  → 左侧列表展示所有可关注日历（可勾选）
  → 右侧列表实时显示已关注列表
  → 默认关注的日历左侧勾选+禁用态、右侧固定
  → 右侧可拖拽排序
  → 点击"确定"保存 / 点击"取消" → 二次确认
```

### 4.4 设为当前版本流程
```
用户 hover 版本标题栏 (478×45)
  → 出现"设为当前版本"按钮 (112×24, 含 aim 图标)
  → 点击按钮
  → 该版本标记为当前版本，标签变为"X.X(当前)"
  → 标题栏按钮变为"当前版本"(84×24) 常驻显示
```

### 4.5 全局版本搜索流程（v1.1 新增）
```
用户点击搜索框
  → 搜索框展开 (200→280px, 200ms ease)
  → 边框变为 accent 色，获得焦点

用户输入关键词（如"白盒"）
  → 300ms 防抖
  → 显示 loading 态（三点脉冲 + "搜索中..."）
  → 接口返回 → 替换为结果面板 (480×auto)
  → 结果按版本日历分组展示
  → 关键词在里程碑名称中高亮显示

用户 ↑↓ 键选择 / 鼠标 hover 结果行
  → 对应行变为 selected 态 (bg-hover)

用户 Enter 键 / 点击某条结果
  → 关闭搜索面板
  → 切换到对应版本日历 Tab
  → 自动滚动定位到对应版本
  → 目标版本标题栏短暂闪烁高亮 (300ms)

用户按 Esc / 点击搜索框外部
  → 关闭搜索面板
  → 搜索框收起 (280→200px)
  → 清空搜索内容

用户点击 clear-btn (✕)
  → 清空输入内容
  → 关闭结果面板
  → 搜索框保持聚焦态
```

## 5. 状态处理

| 状态 | 处理方式 | Figma 参考 |
|------|---------|-----------|
| **无日历空态** | 用户未关注任何版本日历时，使用 `components/empty` 组件 (260×130) 展示空态，包含 empty-pic 图标 (100×100) + 提示文案 (260×22) + 操作按钮区 (250×32) | 弹窗 640×571 无 Tab 态 |
| **有日历无里程碑空态** | 已关注日历但该日历下无里程碑数据，同样使用 `components/empty` 组件展示 | 弹窗 640×571 有 Tab 态 |
| **字数极限情况** | 版本名称/日历名称过长时的截断处理（如"4.13字段比较多的"截断显示），里程碑名称过长时省略号处理（如"4.6版本建模完成最多字数最长情况..."） | Content 区域标注 |
| **滚动吸顶** | 滚动时版本标题栏 (478×45) 吸顶固定，Content 区域含滚动条 (8×101) | 吸顶状态页面 |
| **无数据样式** | 内容区无数据时的展示样式，里程碑列表区域留空 | Content 区域标注 |
| **Tab 过多** | 版本日历 Tab 数量过多时出现 ◄ ► 翻页按钮组 (各 24×24)，selectcard 组件支持横向翻页 | 翻页按钮组标注 |
| **Light 模式** | Light 模式下需要特殊处理渐变遮盖（bg光晕组件在 light 主题下的表现） | 独立标注区域 |
| **搜索无结果** | 展示"未找到相关版本内容"空态图标+文案（v1.1 新增） | — |
| **搜索加载中** | 输入时 300ms 防抖后发起搜索，期间显示 loading 态（v1.1 新增） | — |

## 7. 设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| **浮层而非独立页面** | hover 导航栏标签唤起浮层 | 版本总览是高频查看、低频操作的模块，浮层避免页面跳转、保持上下文 |
| **日历切换用 `*selectcard*` 组件** | selectcard 而非普通 Tab 或下拉选择 | 用户可能频繁切换不同日历，selectcard 更直观高效，支持翻页 |
| **配置用 dialog 弹窗而非页面** | 720×610 dialog 弹窗 | 配置操作相对轻量，不需要整页跳转；dialog 组件提供标准化 head/body/foot 三段式结构 |
| **左右分栏选择模式** | 左侧候选 (340×460) + 右侧已选 (340×460) | 便于同时看到可选项和已选项，类似穿梭框模式 |
| **hover 出现设为当前版本** | 非默认显示 (112×24 按钮) | 设定当前版本是低频操作，常驻显示会增加视觉噪音 |
| **当前版本按钮常驻** | "当前版本"按钮 (84×24) 在当前版本行常驻 | 已是当前版本的行需要明确标识，与 hover 态的"设为当前版本"按钮形成对比 |
| **收起时显示最近时间线** | Milestone Bar 形式 (289×27) | 即使收起也能提供关键时间信息 |
| **默认定位到当前版本** | 自动滚动定位 | 用户最关注当前版本的进度 |
| **右侧支持拖拽排序** | 拖拽而非上下箭头 | 多个日历时拖拽更高效 |
| **Tab 翻页按钮组** | ◄ ► 左右翻页 (各 24×24) | 比折行显示更节省浮层空间，保持紧凑布局 |
| **bg光晕装饰** | 浮层顶部 bg光晕 (146×136) | 提供视觉层次和品质感，Dark/Light 模式有差异处理 |
| **搜索放在吸顶区域** | 吸顶区标题右侧，非独立面板 | 搜索是辅助功能，不应抢占内容区空间；与设置按钮并列，操作入口集中（v1.1 新增） |
| **搜索结果按日历分组** | 分组展示而非平铺 | 用户可能关注多个日历，分组有助于快速定位所属日历上下文（v1.1 新增） |
| **搜索结果点击后跳转** | 关闭搜索 → 切换 Tab → 定位版本 | 一步到位，减少用户手动切换日历+滚动的操作成本（v1.1 新增） |
